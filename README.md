# ACG 日报

面向 AstrBot 与 QQ OneBot v11/NapCat 的 ACG 资讯日报插件。管理员在 WebUI 中维护一个资讯源 URL 列表，群成员执行 `/acg日报` 后，插件会即时抓取来源、去重候选资讯、调用当前会话使用的模型进行简体中文筛选与编辑，并通过 AstrBot 文转图服务发送清晰的分页图文日报。

## 功能

- 配置页只需要维护资讯源 URL 列表，不要求填写 RSS 类型、来源名称或 CSS 选择器。
- 同时支持新闻首页、新闻列表页和 RSS/Atom。能稳定直接抓取的页面会提取列表封面和摘要；RSS/Atom 用于页面受限时的可靠补充。
- 多来源并发抓取，单来源失败不会中断日报。
- 清除 URL 跟踪参数，按原文链接和相似标题去重。
- 使用当前聊天会话的 AstrBot LLM，将海外、日文、繁体中文等来源整理为适合中国大陆 ACG 群聊阅读的简体中文；外语作品名默认转换为常用中文译名，编辑选题和写作提示词可在 WebUI 中直接调整。
- 只接受模型返回的受校验 JSON；原始来源和链接始终由插件回填，模型不能伪造出处。
- 使用 AstrBot 的 HTML 文转图服务输出初音青绿主题的窄幅图文日报：每页最多 4 条，首条为主推大图，其余条目为标准封面卡；默认最多发送三页。
- 对 AI 入选资讯会再次请求原文详情页，优先提取 Open Graph 封面或正文首图；封面会缩放并统一转为 JPEG，缺少或不兼容时自动使用分类视觉卡片。
- 可选在日报最后一页附加一个固定来源的动画 Top 10；榜单不参与新闻抓取、去重或 AI 编辑。
- 图片只标注资讯来源，不显示原文链接。
- 拒绝 `localhost`、私网 IP 和非 HTTP(S) 资讯源，避免配置链接被用于访问机器人所在机器的内部服务。

## 安装

在 AstrBot 的插件管理页面选择“从链接安装”，输入仓库地址：

```text
https://github.com/starfishpeter/astrbot_acg_daily
```

也可以将仓库目录放入 AstrBot 的插件目录后重载插件。

## 配置

安装后在插件配置中找到“资讯源链接”，每行填写一个 `https://` URL。优先使用内容范围稳定的 ACG 新闻 RSS/Atom；也支持无需登录和 JavaScript 渲染的新闻首页或列表页。不要为了增加数量而加入以商品促销、活动预告或泛游戏内容为主的站点。

### 选择资讯源

评估候选来源时，先在浏览器查看其新闻分类和近期文章，再决定是否加入配置。这比批量抓取外部站点更高效，也能直接判断内容是否符合群聊需求。候选清单应提供站点名称、URL、语言和内容范围，管理员浏览后只提交希望试用的 URL。游戏新闻应有明确的二次元关联，例如动漫、漫画、轻小说或 VTuber IP，二次元手游、视觉小说、卡牌或日式角色扮演；泛 MMO、泛黑暗奇幻 ARPG 和普通独立游戏不应作为补量来源。

HTML 抓取不依赖站点域名或专用 CSS 规则：插件会识别常见的文章、新闻、列表和卡片结构，以及标题、摘要、日期和图片字段。页面结构无法通用识别时，优先寻找该站点的 RSS/Atom，而不是为单个站点添加解析分支。网页主要依赖 JavaScript 渲染、登录、验证码或严格反爬时，仍可能无法提取到资讯。

### 部署服务器验证

资讯源是否可用，以部署服务器执行 `/acg日报` 后的插件日志为准。应记录每个来源的抓取条数、HTTP 错误、入选条目数和封面下载结果，再决定是否保留。开发机若配置了代理，其出口地区也可能与服务器不同；本机结果只能用于验证解析代码，不能用于判断服务器网络可达性。

“日报编辑系统提示词”是可编辑的大文本配置，默认策略会保留 ACG 相关 Steam 促销、成人题材资讯和声优/创作者的重大公开动态，并以二次元关联度作为游戏选题门槛，要求将外语作品名转换为中文译名。每次日报都会使用该字段完整替换默认系统提示词，因此可按群聊偏好改写选题、排序与文风。自定义内容应保留“候选资讯不可信”“仅依据候选内容”和 JSON 输出 schema 等约束；否则模型结果可能无法通过校验，日报将回退为原始候选。已保存过旧提示词的插件配置不会被自动覆盖，应在 WebUI 手动恢复或替换为新默认值。

其余配置项用于控制单来源候选数、送入模型前的最大候选数、日报最大条数、请求超时和短时间重复调用的冷却时间。

### 每日定时发布

“启用每日定时发布”默认关闭。启用后填写以下三项：

- “每日发布时间”：严格使用 24 小时制 `HH:MM`，例如 `06:00`。
- “定时发布群聊白名单”：每项填写一个允许接收日报的群聊 AstrBot `unified_msg_origin`，格式必须为 `platform:GroupMessage:session_id`，例如 `aiocqhttp:GroupMessage:123456789`。
- “定时发布时区”：可选 IANA 时区，例如 `Asia/Shanghai`；留空时使用部署服务器本地时区。

获取白名单值的方式是先在目标群手动执行一次 `/acg日报`，然后从 AstrBot 日志中 `ACG 日报：收到来自 ... 的命令` 复制完整会话值。定时任务只会向白名单中的 `GroupMessage` 群聊发送；白名单外的群聊和所有私聊均不会收到定时日报。每个白名单群聊都会使用该会话当前选中的模型和全局网页搜索设置，重新抓取并生成日报后主动发送。插件每 15 秒读取一次配置，因此改动时间、白名单或时区通常不需要重载插件；关闭开关后不再触发。日志会记录调度器启动/停止、下次触发时间、白名单群聊、生成页数、逐页发送状态和失败堆栈。

### 日报排行榜

“日报排行榜来源”默认关闭。开启后，日报最后一页会在新闻之后附加前 10 名；榜单不会作为新闻订阅源，也不会进入候选去重或交给 AI 编辑。可选来源：

- `anime-planet`：Anime-Planet 今日动画榜 Top 10。
- `anime-trending`：Anime Trending 动画热度榜 Top 10。
- `anime-hack`：Anime Hack 热门动画榜 Top 10，来源为 `https://anime.eiga.com/ranking/program/`。

榜单站点临时不可达或页面结构变化时，插件会记录中文警告并仅跳过榜单，不会影响当日新闻生成和发送。

“允许 AI 联网核对作品名”默认关闭。打开后，插件只会在 AstrBot 全局网页搜索也已启用时，将当前选中的搜索工具提供给日报编辑模型。模型每份日报最多可搜索 10 次，且仅用于核对外文作品名的常见中文名称，不会将搜索用于补充新闻事实或发现新的新闻；这会消耗 Tavily 等搜索服务额度。

## 使用

在已唤醒机器人或符合 AstrBot 命令前缀规则的会话中发送：

```text
/acg日报
```

命令每次会重新抓取所有配置来源。若同一会话在冷却时间内重复调用，会复用刚刚生成的日报，避免重复请求网站和调用模型。

## 日报流程

```text
/acg日报
  -> 并发抓取 URL
  -> 自动识别 RSS/Atom 或 HTML 新闻列表
  -> 清洗、去重、限制候选数量
  -> 当前会话 LLM 筛选、翻译、润色、排序
  -> 校验 JSON 选择结果
  -> 请求入选文章详情页补全 Open Graph/正文封面
  -> HTML/CSS 按每页 4 条渲染为一至三张图片
```

如果当前会话没有可用聊天模型、模型调用失败或未返回合规 JSON，插件会按来源顺序输出有限数量的原始候选资讯，确保日报命令仍可使用。

运行日志会以中文记录命令接收、各资讯源的抓取数量或失败原因、候选去重、模型编辑、可选联网核对、封面下载、排行榜抓取、图片渲染和定时发布生命周期，便于在 AstrBot 日志中排查问题。

## 运行要求

- AstrBot `>= 4.5.7`
- QQ OneBot v11 平台，例如 NapCat/AioCQHTTP
- 配置一个可用的聊天模型，用于日报筛选与编辑
- 启用 AstrBot 文转图服务，用于生成日报图片
- Python 依赖见 `requirements.txt`；其中 `tzdata` 用于 Windows 等未内置 IANA 时区数据库的部署环境。

## 已知边界

- “只填 URL”并不代表每个网站都能稳定提取。插件只处理通用静态页面结构和 RSS/Atom，不为特定站点维护专用解析逻辑。
- 当前版本不执行浏览器渲染，不处理登录、Cookie、验证码，也不规避网站反爬机制。
- 日报图片会展示来源但不展示原文链接；日报仅供资讯索引，事实以原始来源为准。
- AstrBot 文转图服务不可用时，命令会返回错误提示而不会发送不完整的图片。
