# ACG 日报

面向 AstrBot 与 QQ OneBot v11/NapCat 的 ACG 资讯日报插件。管理员在 WebUI 中维护一个资讯源 URL 列表，群成员执行 `/acg日报` 后，插件会即时抓取来源、去重候选资讯、调用当前会话使用的模型进行简体中文筛选与编辑，并通过 AstrBot 文转图服务发送一张图文日报。

## 功能

- 配置页只需要维护资讯源 URL 列表，不要求填写 RSS 类型、来源名称或 CSS 选择器。
- 同时支持新闻首页、新闻列表页和 RSS/Atom。能稳定直接抓取的页面会提取列表封面和摘要；RSS/Atom 用于页面受限时的可靠补充。
- 多来源并发抓取，单来源失败不会中断日报。
- 清除 URL 跟踪参数，按原文链接和相似标题去重。
- 使用当前聊天会话的 AstrBot LLM，将海外、日文、繁体中文等来源整理为适合中国大陆 ACG 群聊阅读的简体中文。
- 只接受模型返回的受校验 JSON；原始来源和链接始终由插件回填，模型不能伪造出处。
- 使用 AstrBot 的 HTML 文转图服务输出一张编辑部画报风格的图文日报：首条主推大图，其余条目双列封面卡。
- 对 AI 入选资讯会再次请求原文详情页，优先提取 Open Graph 封面或正文首图；缺少封面时自动使用分类视觉卡片。
- 图片只标注资讯来源，不显示原文链接。
- 拒绝 `localhost`、私网 IP 和非 HTTP(S) 资讯源，避免配置链接被用于访问机器人所在机器的内部服务。

## 安装

在 AstrBot 的插件管理页面选择“从链接安装”，输入仓库地址：

```text
https://github.com/starfishpeter/astrbot_acg_daily
```

也可以将仓库目录放入 AstrBot 的插件目录后重载插件。

## 配置

安装后在插件配置中找到“资讯源链接”。每行填写一个 `https://` URL。建议同时保留可稳定提取的新闻页面与官方 RSS，前者通常有列表摘要，后者在页面结构变化或访问受限时更可靠：

```text
# 已验证的新闻页面：列表页直接有封面和摘要
https://animeuknews.net/
https://www.chuapp.com/

# 已验证的 RSS：页面受限时的可靠补充
https://myanimelist.net/rss/news.xml
https://ln-news.com/feed
https://animecorner.me/feed/
https://hobby.dengeki.com/feed/
```

`Anime UK News` 与 `触乐` 的页面结构会直接给出缩略图和摘要，适合图片日报。`MyAnimeList`、`Anime Corner` 与 `电击 Hobby` 建议使用 RSS；入选后插件仍会访问原文详情页补全封面。实测中 `Anime Herald`、`AnimeAnime` 的响应不够稳定，Anime News Network 在部分服务器网络下会返回 `403`，因此暂不建议作为默认来源。网页主要依赖 JavaScript 渲染、登录、验证码或严格反爬时，仍可能无法提取到资讯。

其余配置项用于控制单来源候选数、送入模型前的最大候选数、日报最大条数、请求超时和短时间重复调用的冷却时间。

“允许 AI 联网核对作品名”默认关闭。打开后，插件只会在 AstrBot 全局网页搜索也已启用时，将当前选中的搜索工具提供给日报编辑模型。模型最多进行一次搜索，且仅用于核对外文作品名的常见中文名称，不会将搜索用于补充新闻事实或发现新的新闻；这会消耗 Tavily 等搜索服务额度。

## 使用

在已唤醒机器人或符合 AstrBot 命令前缀规则的会话中发送：

```text
/acg日报
```

命令每次会重新抓取所有配置来源。若同一会话在冷却时间内重复调用，会复用刚刚生成的日报，避免重复请求网站和调用模型。

## 日报流程

```text
/acg日报
  -> 并发抓取 URL
  -> 自动识别 RSS/Atom 或 HTML 新闻列表
  -> 清洗、去重、限制候选数量
  -> 当前会话 LLM 筛选、翻译、润色、排序
  -> 校验 JSON 选择结果
  -> 请求入选文章详情页补全 Open Graph/正文封面
  -> HTML/CSS 渲染为单张图片
```

如果当前会话没有可用聊天模型、模型调用失败或未返回合规 JSON，插件会按来源顺序输出有限数量的原始候选资讯，确保日报命令仍可使用。

运行日志会以中文记录命令接收、各资讯源的抓取数量或失败原因、候选去重、模型编辑、可选联网核对、封面下载和图片渲染，便于在 AstrBot 日志中排查问题。

## 运行要求

- AstrBot `>= 4.5.7`
- QQ OneBot v11 平台，例如 NapCat/AioCQHTTP
- 配置一个可用的聊天模型，用于日报筛选与编辑
- 启用 AstrBot 文转图服务，用于生成日报图片
- Python 依赖见 `requirements.txt`

## 开发测试

```bash
python -m unittest discover -s tests -v
```

测试覆盖 RSS/Atom 与普通 HTML 列表抽取、MyAnimeList/ANN/触乐页面适配、详情页 Open Graph 封面发现、去重与来源轮转、链接清洗、内网地址拒绝、模型 JSON 校验和图片日报 HTML 转义。

## 已知边界

- “只填 URL”并不代表每个网站都能稳定提取。对通用静态新闻页可自动处理；特殊热门站点可在后续版本添加内部专用适配器，配置页面无需改变。
- 当前版本不执行浏览器渲染，不处理登录、Cookie、验证码，也不规避网站反爬机制。
- 日报图片会展示来源但不展示原文链接；日报仅供资讯索引，事实以原始来源为准。
- AstrBot 文转图服务不可用时，命令会返回错误提示而不会发送不完整的图片。
