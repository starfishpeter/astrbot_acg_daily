# Changelog

## Unreleased

- 新增单独的 Tavily 页面提取资讯源列表：每个公开 URL 通过 AstrBot 内置 `tavily_extract_web_page` 独立读取为不可信候选材料，不使用关键词搜索或插件 API Key；普通 RSS/HTML 来源保持原有抓取路径。Tavily 列表页会按提取内容中的文章链接拆分候选；返回完整 RSS/Atom/RDF 时会按订阅条目解析，不再将整份订阅压缩为一条候选。
- 移除可覆盖安全和输出契约的日报编辑系统提示词配置、定时发布时区配置和译名网页搜索兜底；编辑固定使用内置提示词，定时按 AstrBot 服务器本地时区触发，译名核对仅使用 Bangumi 候选。
- 加强编辑、榜单和 Bangumi 外部 JSON 的类型校验，跳过异常条目，避免布尔值或空值被误当作有效标识和作品信息。
- 资讯源地址校验改用标准库的全局可路由地址判定，拒绝运营商级 NAT 等非公网地址；日报卡片模板改为每次渲染仅加载一次。
- 编辑提示词、WebUI 默认值、无工具重试和解析器统一了日报 JSON 字段类型约束，并增加提示词样例可被当前解析器接受的回归测试。
- 定时发布会将旧的 `aiocqhttp:GroupMessage:群号` 自动解析为唯一运行中的 QQ 平台实例 ID；多实例部署可在白名单填写 `平台ID:GroupMessage:群号`，不匹配时会在抓取和渲染前失败。

## 0.5.1 - 2026-07-21

- 批量译名核对新增可选 Bangumi Access Token：优先查询动画、书籍/漫画和游戏词条，向编辑模型返回多个 `name_cn` 候选以结合新闻上下文判断同名作品。
- 网页搜索改为 Bangumi 未返回中文候选时的兜底；令牌仅用于 Bangumi Authorization 请求头，不会进入提示词、工具结果、日志或日报图片。

## 0.5.0 - 2026-07-21

- 日报一图流改为填满渲染宽度的横向资讯卡：封面置左、正文置右，封面完整显示而不再按 `cover` 裁剪；资讯卡统一尺寸，排行榜保持从 1 到 10 的纵向排列。
- 编辑候选移除未使用的来源与发布时间字段，并将摘要输入压缩至 160 字；日志会记录候选、榜单和实际输入字符数。
- 联网译名核对改为一次模型可见的批量工具调用：至多 20 个作品名分为最多 3 个并发搜索组，搜索结果压缩后统一交回模型，避免逐标题十轮工具循环；搜索组兼容 AstrBot 的各内置搜索提供商。

## 0.4.9 - 2026-07-21

- 插件重载或卸载时，已在运行的旧日报任务即使随后完成，也会在发送前丢弃结果，避免旧任务与新命令同时发送多张日报。

## 0.4.8 - 2026-07-21

- 新增 WebUI “编辑慢请求诊断阈值（秒）”，默认 600 秒；达到阈值仅记录 Agent 阶段日志并继续等待，设为 0 可关闭该诊断日志。

## 0.4.7 - 2026-07-21

- 编辑模型等待超过 120 秒时改为仅记录中文阶段诊断并继续等待，不再取消仍在处理搜索结果或生成日报的 Agent；保留调用中断时的任务清理。

## 0.4.6 - 2026-07-21

- 为带联网工具的日报编辑和无工具重试增加 120 秒总等待上限；工具循环超时会记录中文阶段诊断、取消等待并自动进入无工具重试，避免模型或工具调用无限挂起。

## 0.4.5 - 2026-07-21

- 新增 WebUI “日报编辑模型”选择器；留空时跟随当前会话模型，选择后正常编辑和无工具重试均固定使用指定模型，方便切换模型排查问题。
- 精简默认编辑提示词为核心二次元群友视角：番剧、漫画和轻小说优先，二次元游戏其次，并保留中文译名、有限联网核对译名和 JSON 输出等必要约束。
- 排行榜 Top 10 与新闻候选合并到同一次日报编辑调用，新闻和榜单标题共用最多 10 次联网名称核对；榜单翻译不完整或失败时跳过榜单，避免外文标题直出。

## 0.4.4 - 2026-07-20

- 调整日报模板在 AstrBot 默认文转图视口中的缩放比例，减少一图流两侧空白。
- 带联网工具的编辑模型未返回合规 JSON 时，自动使用同一模型进行一次无工具重试；两次失败后跳过未经翻译和筛选的原始候选，不再回退直出。

## 0.4.3 - 2026-07-20

- 恢复日报一图流：所有资讯和可选 Top 10 均渲染为单张长图，只发送一次图片消息；无封面资讯改为紧凑提示区。
- 封面 JPEG 规范化容忍可恢复的轻微截断图片，避免 Debian 上 Pillow 的严格解码将可用封面全部退化为占位卡。

## 0.4.2 - 2026-07-20

- 定时发布群聊白名单改为可直接填写 QQ 群号；仍兼容完整的 `aiocqhttp:GroupMessage:群号` 会话格式。

## 0.4.1 - 2026-07-20

- 新增 WebUI 可配置的每日定时发布：可指定 `HH:MM`、IANA 时区和仅接受 `GroupMessage` 的群聊白名单；插件生命周期会管理调度器并记录触发、生成、发送和失败日志，不依赖 v4.14.0 后的实验性 FutureTask 接口。
- 新增可选的内置动画排行榜来源：Anime-Planet、Anime Trending 和 Anime Hack；选中的 Top 10 附在日报最后一页，抓取失败时不影响新闻日报。
- HTML 新闻页改用通用文章、列表和卡片结构抽取，不再按站点域名维护专用 CSS 选择器或标题过滤规则。
- 日报编辑明确优先保留作品发布、制作、播出、版权和行业动态，排除纯周边、抽奖、快闪、促销和无新增事实的例行内容。
- 文档改为浏览器先行的选源流程：先由管理员浏览候选站点，再将希望试用的 URL 加入配置；页面无法通用解析时优先使用其 RSS/Atom。
- 新增 WebUI 可编辑的日报编辑系统提示词；默认允许 ACG 相关 Steam 促销、成人题材资讯，以及声优和创作者的重大公开动态。
- 日报默认最多收录 12 条资讯并输出一张长图；默认标题规则要求使用中文译名，中文社群长期直接使用的 Fate、VTuber、Steam、hololive 等名称除外。
- 外文作品名的联网名称核对上限从每份日报 1 次提升至 10 次，仍禁止以搜索结果补充新闻事实。
- 游戏选题改以二次元关联度为首要门槛，默认降低泛 MMO、泛黑暗奇幻 ARPG 和普通独立游戏的优先级。
- 详情页和封面 CDN 请求各增加一次短暂失败重试；补充 GNN WebP/PNG 封面与无图 ANN 页面占位的回归测试。

## 0.4.0 - 2026-07-20

- 重做日报图片为独立主题页和资讯卡模板；每页最多 4 条资讯，以两页发送 8 条日报，提升 QQ 内阅读字号。
- 封面在插入 HTML 前统一缩放并重编码为 JPEG，避免海外服务器 Chromium 对大尺寸或特殊格式 data URI 出现破图。
- 封面原始文件上限调整为 4 MiB；无可用封面时使用可读的分类视觉卡片。

## 0.3.0 - 2026-07-20

- 日报图片更新为编辑部画报布局：首条主推大图，后续资讯使用双列封面卡。
- AI 入选资讯会抓取原文详情页，优先使用 Open Graph 或正文首图；无图时使用分类视觉卡片。
- 改进新闻页面的列表图片提取；候选列表按来源轮转，减少单一站点占满日报的情况。
- 更新默认来源建议，优先使用内容稳定的新闻页面和官方 RSS。

## 0.2.1 - 2026-07-20

- 抓取未返回条目的资讯源时自动重试一次。
- 抓取失败日志增加响应类型和大小，便于区分空订阅与站点拦截页面。
- 增加封面下载、AstrBot 文转图和图片发送各阶段的中文运行日志。

## 0.2.0 - 2026-07-20

- 日报输出从 QQ 合并转发改为 AstrBot HTML 文转图生成的单张图文长图。
- 日报最多收录 8 条资讯；图片显示来源但不显示原文链接。
- 支持从 RSS、Atom、RDF 和普通列表页提取封面图；无法取得封面时使用分类文字卡片。
- 增加封面下载的私网地址、重定向、格式和大小限制。
- 改进二次元内容筛选，排除泛游戏、科技和成人向资讯。
- 支持 RSS 1.0/RDF 订阅源，并补充图片日报与封面提取测试。

## 0.1.1 - 2026-07-20

- 增加中文运行日志和可选的网页搜索核对功能。

## 0.1.0 - 2026-07-20

- 初始版本：从可配置资讯源抓取 ACG 新闻，经 AI 筛选和整理后发送日报。
