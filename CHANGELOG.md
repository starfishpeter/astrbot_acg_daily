# Changelog

## 0.4.4 - 2026-07-20

- 调整日报模板在 AstrBot 默认文转图视口中的缩放比例，减少一图流两侧空白。
- 带联网工具的编辑模型未返回合规 JSON 时，自动使用同一模型进行一次无工具重试；两次失败后跳过未经翻译和筛选的原始候选，不再回退直出。

## 0.4.3 - 2026-07-20

- 恢复日报一图流：所有资讯和可选 Top 10 均渲染为单张长图，只发送一次图片消息；无封面资讯改为紧凑提示区。
- 封面 JPEG 规范化容忍可恢复的轻微截断图片，避免 Debian 上 Pillow 的严格解码将可用封面全部退化为占位卡。

## 0.4.2 - 2026-07-20

- 定时发布群聊白名单改为可直接填写 QQ 群号；仍兼容完整的 `aiocqhttp:GroupMessage:群号` 会话格式。

## 0.4.1 - 2026-07-20

- 新增 WebUI 可配置的每日定时发布：可指定 `HH:MM`、IANA 时区和仅接受 `GroupMessage` 的群聊白名单；插件生命周期会管理调度器并记录触发、生成、发送和失败日志，不依赖 v4.14.0 后的实验性 FutureTask 接口。
- 新增可选的内置动画排行榜来源：Anime-Planet、Anime Trending 和 Anime Hack；选中的 Top 10 附在日报最后一页，抓取失败时不影响新闻日报。
- HTML 新闻页改用通用文章、列表和卡片结构抽取，不再按站点域名维护专用 CSS 选择器或标题过滤规则。
- 日报编辑明确优先保留作品发布、制作、播出、版权和行业动态，排除纯周边、抽奖、快闪、促销和无新增事实的例行内容。
- 文档改为浏览器先行的选源流程：先由管理员浏览候选站点，再将希望试用的 URL 加入配置；页面无法通用解析时优先使用其 RSS/Atom。
- 新增 WebUI 可编辑的日报编辑系统提示词；默认允许 ACG 相关 Steam 促销、成人题材资讯，以及声优和创作者的重大公开动态。
- 日报默认最多收录 12 条资讯并输出一张长图；默认标题规则要求使用中文译名，中文社群长期直接使用的 Fate、VTuber、Steam、hololive 等名称除外。
- 外文作品名的联网名称核对上限从每份日报 1 次提升至 10 次，仍禁止以搜索结果补充新闻事实。
- 游戏选题改以二次元关联度为首要门槛，默认降低泛 MMO、泛黑暗奇幻 ARPG 和普通独立游戏的优先级。
- 详情页和封面 CDN 请求各增加一次短暂失败重试；补充 GNN WebP/PNG 封面与无图 ANN 页面占位的回归测试。

## 0.4.0 - 2026-07-20

- 重做日报图片为独立主题页和资讯卡模板；每页最多 4 条资讯，以两页发送 8 条日报，提升 QQ 内阅读字号。
- 封面在插入 HTML 前统一缩放并重编码为 JPEG，避免海外服务器 Chromium 对大尺寸或特殊格式 data URI 出现破图。
- 封面原始文件上限调整为 4 MiB；无可用封面时使用可读的分类视觉卡片。

## 0.3.0 - 2026-07-20

- 日报图片更新为编辑部画报布局：首条主推大图，后续资讯使用双列封面卡。
- AI 入选资讯会抓取原文详情页，优先使用 Open Graph 或正文首图；无图时使用分类视觉卡片。
- 改进新闻页面的列表图片提取；候选列表按来源轮转，减少单一站点占满日报的情况。
- 更新默认来源建议，优先使用内容稳定的新闻页面和官方 RSS。

## 0.2.1 - 2026-07-20

- 抓取未返回条目的资讯源时自动重试一次。
- 抓取失败日志增加响应类型和大小，便于区分空订阅与站点拦截页面。
- 增加封面下载、AstrBot 文转图和图片发送各阶段的中文运行日志。

## 0.2.0 - 2026-07-20

- 日报输出从 QQ 合并转发改为 AstrBot HTML 文转图生成的单张图文长图。
- 日报最多收录 8 条资讯；图片显示来源但不显示原文链接。
- 支持从 RSS、Atom、RDF 和普通列表页提取封面图；无法取得封面时使用分类文字卡片。
- 增加封面下载的私网地址、重定向、格式和大小限制。
- 改进二次元内容筛选，排除泛游戏、科技和成人向资讯。
- 支持 RSS 1.0/RDF 订阅源，并补充图片日报与封面提取测试。

## 0.1.1 - 2026-07-20

- 增加中文运行日志和可选的网页搜索核对功能。

## 0.1.0 - 2026-07-20

- 初始版本：从可配置资讯源抓取 ACG 新闻，经 AI 筛选和整理后发送日报。
